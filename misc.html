App architecture - Model, View, Controller (MVC)

We expect fellows to understand how to use MVC pattern to design an iOS app. Understand that MVC is a very common app architecture but there are other options used by the industry, e.g MVVM. MVC is heavily used by Apple. Components of MVC:

Model: the data needed by your app

View: visual aspects of the user interface

Controller: the messenger (glue) between the views and the models


REST-ful APIs, JSON, and AJAX

FSW.5 Deep knowledge of and familiarity with REST-ful APIs. Build a frontend application that queries remote APIs to render information to the user.

FSW.5.a Know HTTP and the request-response cycle.

FSW.5.b Utilize GET, POST, PATCH, and DELETE requests and the difference between them. Utilize CRUD requests to create, update, or delete information from an API.

FSW.5.c Utilize various libraries/services to make AJAX requests to APIs (including fetch, axios, XMLHttpRequest).

FSW.5.d Understand API routing, wildcards, and formatting appropriate requests in order to query an API in Postman or the browser to get a desired response.

FSW.5.e Able to navigate new API documentation to discern how an API might be useful in a project.

FSW.5.e Know how to handle, parse, and represent data received in JSON format.


HTTP/APIs

We expect fellows to be abke to start making requests to the internet and web APIs. Fellows should have a basic understanding of how the internet works. The basic understanding concepts includes:

Understand how the internet works

Know the following HTTP verbs: POST, GET, PUT, DELETE

Be familiar with response status codes: 100 -> 500

Understand what is a RESTFul API

Know how to use URLSession along with URLRequest to make requests to external web APIs

We expect fellows to know how to manage API keys and keeping them safe.
