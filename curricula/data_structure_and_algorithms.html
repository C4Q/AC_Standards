# C4Q - Access Code ## Data Structures and Algorithms Standards ## Executive Summary Data Structures & Algorithms (DSA) are the building blocks that allow programmers to code and think at a higher level. Data structures are unique ways of organising and collecting data so that we can perform operations in an effective way. Algorithms are a set of instructions that solve hard problems in an efficient manner. Understanding DSA is critical for entry level interviews. DSA is usually technology agnostic so it instills a strong computer science mindset. ## Goals - Students should be able to build and apply important Data Structures used in the real world. - Students should be able to design and apply correct Algorithms for programming problems. - Students should get a basic sense of "abstract data structures" and how the flow of data occurs in applications. - Students should begin to understand efficiency of code and learn to code beyond the "brute force" route. - Students should be able to problem solve at a high level during whiteboarding interviews. ## Topics 1. Native & Abstract Data Types 2. Big(O) Notation 3. Lists 4. Stacks & Queues 5. Hash Table 6. Recursion + Divide & Conquer 7. Sorting Algorthims (Bubble / Quick / Merge) 8. Binary Search Tree 9. Search Algorithms (Binary Search / Depth First / Breadth First) 10. Graphs & Dijkstra’s algorithm ## Curriculum ### 1) Native & Abstract Data Types
Native Data Types
  • Understand that data types are the foundation of all code
  • Different ways of storing information drastically changes the way you approach code, example: Int 1 compared to String "1"
  • Native data types are in every language: Integer, Float, String, Char
Abstract Data Types
  • Student should have basic OOP understanding and coding classes
  • Understand that native data types and how you associate them is the foundation of all abstract data types
  • Show examples of association between data, example: linking lists and graphs
  • Understand that what a "Node" is and how you define it in code
### 2) Big(O) Notation
Runtime Complexity
  • Student should have encountered nested loops
  • Understand the difference between O(1) and O(n)
  • Understand the relationship between data collection size and run time
  • Show and explain big O graph
### 3) Lists
Singly Linked List
  • Student should understand how to build a Node class
  • Understand the relationship between nodes
  • Implement SLL class with Head only
  • Implement: push(), pop(), find()
  • Understand the run time for each method
Doubly Linked List
  • Implement DLL class with Head and Tail pointers
  • Implement: push(), pop(), find(), removeAt()
  • Understand the run time for each method
### 4) Stacks & Queues
Stacks
  • LIFO
  • Student should understand real world use cases
  • Implement: push(), pop(), isEmpty()
  • Understand the run time for each method
Queues
  • FIFO
  • Student should understand real world use cases
  • Implement: Enqueue(), Dequeue(), isEmpty()
  • Understand the run time for each method
### 5) Hash Tables
Concept
  • Student should understand dictionary
  • Key and Value Mapping
  • Real world use cases
  • Common methods and run times
Implementation
  • Student should understand the reason for a Hash Method
  • How to get O(1) insert and find
  • Student should understand collision
  • Student should understand Best Case vs Worst Case runtimes
### 6) Recursion + Divide & Conquer
Recursion Basics
  • Student should implement Factorial function
  • Student should implement Fibonnaci
  • Student should understand call stack and returns
Divide and Conquer
  • Student should try brute force search on an Array
  • Student should understand worst case for brute force search
  • Student should understand binary search
  • Student should understand worst case for binary search
  • Student should implement binary search
### 7) Sorting Algorthims (Bubble / Quick / Merge)
Sorting Premise and Bubble Sort
  • Student should understand why we sort and why it's useful
  • Real world examples of sorting
  • Compare different sorting algorithms and runtimes
  • Implement Bubble Sort
Quick Sort
  • Run time
  • Implementation
Merge Sort
  • Run time
  • Implementation
### 8) Binary Search Tree
Trees
  • Student should understand abstraction of a tree data type
  • Should be able to easily implement using Nodes
  • Understand use cases
Binary Search Tree
  • Student should understand rules of a BST
  • Student should understand use case of a BST
  • Student should implement insert() and find()
  • Min() and Max()
### 9) Search Algorithms
Depth First / Breadth First
  • Implement Depth First Search
  • Implement Breadth First Search
### 10) Graphs & Dijkstra’s Algorithm
Graphs
  • Student should understand abstraction of a graph data type
  • Should be able to easily implement using Nodes
  • Understand use cases
Dijkstra’s Algorithm
  • Student should understand concept of shortest path algorithm
  • Understand vertex
  • Understand use cases, example: Google Maps, MTA Route to Destination
  • Implement Dijkstra's Algorithm
